# 页面笔记系统 V1 PRD

## 1. 背景

当前插件已经支持：

- 页面高亮
- 批注
- 侧边栏查看当前页与全部高亮
- 管理页集中整理
- Markdown 导出

但在真实使用中，用户在阅读高信息密度网页、以及使用 ChatGPT、Gemini、DeepSeek 等对话型大模型页面时，除了高亮之外，还需要记录：

- 当前想到的观点
- 后续待追问的问题
- 信息之间的关联
- 临时草稿和整理思路

仅靠高亮和批注不足以承载这些内容，因此需要引入“页面笔记”能力。

## 2. 产品目标

V1 的目标是：

- 在“当前页面”中提供低摩擦的页面级笔记能力
- 支持输入长段文字
- 自动保存
- 刷新或重开侧边栏后仍保留
- 对话型大模型页面按完整对话 URL 分开保存
- 不破坏现有高亮、删除、批量选择等稳定功能

## 3. 非目标

V1 明确不做：

- 独立“页面笔记”主 tab
- 富文本编辑
- Markdown 编辑器
- AI 自动整理
- 多篇笔记
- 跨页面统一笔记本
- 云同步
- 账号系统
- 付费体系
- 笔记导出 UI

说明：
导出系统和商业化系统会在第二大任务中处理，但 V1 的数据结构必须为后续导出预留好能力。

## 4. 核心用户场景

### 场景 A：对话型大模型页面快速记录

用户在 ChatGPT、Gemini、DeepSeek 等某个对话页面中高亮了几段关键内容，希望立刻记下：

- 这段内容的可信度如何
- 下一轮要问什么
- 哪几段可以放进后续文章

用户希望：

- 不离开当前页面
- 不切换复杂页面
- 可以直接输入长段笔记
- A 对话和 B 对话的笔记严格分开

### 场景 B：网页阅读时做页面级摘要

用户在一篇文章或资料页中边看边记，希望留下：

- 本页总结
- 关键观点
- 待核查问题

用户希望笔记作为“当前页工作区”的一部分存在，而不是额外跳到一个新的系统中。

## 5. 方案结论

V1 不新增第三个主 tab。

侧边栏继续保留两个主 tab：

- 当前页面
- 全部高亮

在“当前页面”tab 中新增两个工作模块：

- 当前页高亮
- 当前页笔记

其中“当前页笔记”是页面级工作区，不是独立系统。

## 6. 功能范围

### 6.1 当前页面笔记

支持：

- 显示“当前页笔记”模块
- 模块默认收起
- 显示当前页是否已有笔记
- 展开后显示大尺寸输入区
- 自动保存
- 显示保存状态
- 显示最后更新时间
- 显示字数

### 6.2 数据归属

页面笔记必须按页面 URL 归属。

V1 的归属规则：

- 默认使用完整 URL 作为页面笔记主键
- 对话型大模型页面按完整对话 URL 分开存储
- 不允许把同站点不同对话页笔记归并到同一条记录中

## 7. 管理页要求

V1 管理页不提供完整笔记编辑能力，只做轻量展示：

- 页面条目可显示“是否有笔记”
- 可显示笔记字数或“有笔记”标识
- 不在管理页做大输入框编辑

## 8. 成功标准

以下全部满足，视为 V1 完成：

- 普通网页可创建并保存页面笔记
- 对话型大模型页面的不同对话页笔记不会串页
- 页面刷新后笔记仍保留
- 侧边栏关闭后再次打开，笔记仍可读取
- 高亮主链路无回归
- 删除高亮、批量删除、导出 Markdown 仍正常
- 管理页能看出哪些页面有笔记

## 9. 风险与约束

- 侧边栏宽度有限，必须保证大段输入体验，不适合堆太多附属按钮
- 当前高亮和当前页笔记共处一页，必须避免相互抢占空间
- 不允许在实现中把页面笔记和高亮数组混存，否则后续导出和商业化会返工

## 10. V1 之后的自然演进

本次实现后，应自然支持后续扩展：

- 高亮 + 笔记一起导出
- 笔记搜索
- 有笔记页面筛选
- 付费导出目标
- AI 整理
