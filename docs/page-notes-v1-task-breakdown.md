# 页面笔记系统 V1 开发任务拆解

## 1. 交付目标

本轮开发完成后，应能做到：

- 侧边栏“当前页面”内有页面笔记工作区
- 支持长文本输入
- 自动保存
- 刷新后保留
- 对话型大模型页面严格分开
- 管理页显示哪些页面有笔记
- 现有高亮主链路无回归

## 2. 给 AI 的开发范围约束

每次交给 AI 时，应明确：

- 不新增第三个主 tab
- 不做富文本
- 不做导出功能
- 不做付费功能
- 不重构 `content.js` 的高亮主逻辑
- 不修改已经稳定的高亮删除/批量选择交互，除非直接相关

## 3. 实施阶段

### 阶段 1：共享存储层

目标：

- 新增 `note-repo.js`
- 新增 `page-note-service.js`

完成标准：

- 能按完整 URL 读写页面笔记
- 能返回标准化 `PageNoteRecord`
- 能计算字数和更新时间

建议文件：

- `note-repo.js`
- `page-note-service.js`

### 阶段 2：侧边栏接入页面笔记工作区

目标：

- 在“当前页面”tab 中增加“当前页笔记”模块
- 支持展开/收起
- 支持 textarea 输入
- 支持自动保存状态

主要修改文件：

- `sidepanel.html`
- `sidepanel.css`
- `sidepanel.js`

完成标准：

- 打开当前页面可看到笔记模块
- 输入内容可自动保存
- 显示保存状态
- 页面信息卡显示本页笔记摘要

### 阶段 3：对话型大模型页面验证

目标：

- 确保 A/B 两个对话 URL 的笔记不串

完成标准：

- A 页记的内容不会出现在 B 页
- 刷新后仍正确读取各自笔记

### 阶段 4：管理页轻量展示

目标：

- 管理页显示页面是否有笔记

主要修改文件：

- `options.js`
- `options.css`

完成标准：

- 页面卡片可显示“有笔记”或字数信息
- 不新增大段编辑区

### 阶段 5：回归验证

目标：

- 验证高亮功能无回归

完成标准：

- 当前页面高亮正常
- 全部高亮正常
- 删除本页正常
- 管理页多选正常
- 页面笔记正常

## 4. AI 执行建议顺序

建议按以下顺序给 AI：

1. 先实现 `note-repo.js`
2. 再实现 `page-note-service.js`
3. 再接入 `sidepanel` UI
4. 最后接管理页显示

不要一上来让 AI 同时改 6 个文件并设计全部交互。

## 5. 每轮 AI 任务模板

推荐模板：

```text
任务目标：
实现页面笔记系统 V1 的某一个阶段。

本轮只做：
1. ...
2. ...

不要做：
1. 不新增第三个主 tab
2. 不做导出
3. 不做付费
4. 不做富文本

必须保证：
1. 页面笔记独立于高亮存储
2. 对话型大模型页面按完整 URL 隔离
3. 不破坏当前高亮/删除/批量选择

完成后请说明：
1. 改了哪些文件
2. 核心数据结构是什么
3. 手动应如何验证
```

## 6. 风险提示

- 侧边栏是高频使用区，不适合让 AI 一次性大改布局
- 如果 AI 未理解“笔记属于当前页面工作区，而不是独立 tab”，很容易跑偏
- 如果 AI 偷懒把笔记混进高亮存储，会导致后续返工

## 7. 开发完成的判定条件

同时满足以下条件才算 V1 完成：

- 当前页面笔记可编辑
- 自动保存正常
- 刷新后保留
- 对话型大模型页面不串页
- 管理页可见笔记状态
- 高亮主流程未回归
