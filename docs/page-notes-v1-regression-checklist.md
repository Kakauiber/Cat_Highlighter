# 页面笔记系统 V1 回归测试清单

## 1. 基础页面笔记

- 打开普通网页，侧边栏“当前页面”可看到“当前页笔记”模块
- 展开笔记模块后，textarea 可输入长段文字
- 输入后显示 `保存中...`
- 保存完成后显示 `已保存`
- 关闭侧边栏再打开，内容仍存在
- 刷新页面后，内容仍存在

## 2. 对话型大模型页面隔离

- 打开 ChatGPT、Gemini 或 DeepSeek 的某个对话页 A，输入一段页面笔记
- 打开同站点另一对话页 B，确认初始不显示 A 的内容
- 在 B 再输入另一段笔记
- 来回切换 A/B，确认各自内容独立
- 刷新 A/B 后，内容仍各自独立

## 3. 当前页面高亮回归

- 当前页面高亮数量显示正常
- 当前页面删除单条高亮正常
- 当前页面删除本页高亮正常
- 当前页面复制本页高亮正常
- 当前页面笔记存在时，不影响高亮交互

## 4. 全部高亮回归

- 全部高亮列表正常显示
- 搜索高亮内容正常
- 多选模式可正常选中
- 页面级选择框可正常工作
- 批量删除正常
- 导出所选正常

## 5. 管理页回归

- 管理页正常加载页面列表
- 管理页可显示哪些页面有笔记
- 管理页原有高亮删除功能正常
- 管理页多选功能正常
- 管理页批量删除后，原网页高亮同步消失

## 6. 笔记状态展示

- 当前页面信息卡能正确显示“笔记：暂无”或“笔记：已记录”
- 管理页页面项能正确显示“有笔记”或字数信息
- 空笔记页面不会误显示为有笔记

## 7. 异常场景

- 输入很长一段文字后保存不应卡死
- 保存失败时不应清空 textarea 内容
- 在收起/展开笔记模块后内容不丢失
- 切换页面后不应显示上一页笔记

## 8. 最终通过标准

如果以下全部满足，可视为可进入下一阶段：

- 页面笔记主功能稳定
- 对话型大模型页面场景稳定
- 高亮功能无明显回归
- 管理页无明显回归
