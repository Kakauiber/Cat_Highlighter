# 页面笔记系统 V1 UI / 交互规格

## 1. 总体原则

- 不新增第三个主 tab
- 页面笔记属于“当前页面”工作区
- 侧边栏以低摩擦记录为优先
- 输入区必须适合长段输入
- 不影响现有高亮模块的主路径

## 2. 侧边栏信息架构

主 tab 保持：

- 当前页面
- 全部高亮

“当前页面”tab 内部结构：

1. 页面信息卡
2. 当前页高亮模块
3. 当前页笔记模块

## 3. 当前页面布局

### 3.1 页面信息卡

保留现有内容：

- 页面标题
- 页面 URL
- 高亮数量
- 页面级操作按钮

新增一条轻量状态：

- `笔记：暂无`
- `笔记：已记录`
- 或 `笔记：324 字`

说明：
这条状态只用于让用户快速知道“本页有没有笔记”。

### 3.2 当前页高亮模块

- 默认展开
- 保持现有样式和交互不变
- 在笔记模块展开时可以保持存在，但优先级低于笔记输入区

### 3.3 当前页笔记模块

#### 默认状态

- 默认收起
- 仅显示模块标题和摘要信息

摘要信息建议文案：

- `暂无笔记`
- `已记录 126 字`
- `最后更新：刚刚`

#### 展开状态

展开后展示：

- 模块标题栏
- 保存状态
- 更新时间
- 大尺寸 textarea
- 字数信息

textarea 设计要求：

- 至少占据当前 tab 的主要可视高度
- 支持多行长文本输入
- 输入时不应频繁跳动

## 4. 建议交互

### 4.1 模块标题栏

建议字段：

- 标题：`当前页笔记`
- 保存状态：`已保存 / 保存中 / 保存失败`
- 折叠开关

### 4.2 自动保存

触发方式：

- 用户输入后延迟保存
- 失焦时补一次保存

建议状态：

- 初始：不显示或显示 `暂无修改`
- 输入中：`保存中...`
- 保存成功：`已保存`
- 保存失败：`保存失败，请重试`

### 4.3 折叠行为

建议：

- 笔记模块默认收起
- 用户点开后保持当前展开状态，直到关闭侧边栏
- 同一轮使用过程中，不要频繁自动重置展开状态

## 5. 管理页展示

V1 管理页不做笔记编辑，只做识别：

- 页面项增加 `📝` 或 `有笔记`
- 可显示字数，例如 `📝 126字`

不建议 V1 就做：

- 管理页内直接编辑大段笔记
- 笔记单独筛选页
- 笔记全文预览

## 6. 空状态与异常状态

### 6.1 笔记空状态

- `暂无笔记，随手记下你的想法或待追问点`

### 6.2 保存失败状态

- 保留 textarea 内容
- 状态提示为 `保存失败`
- 提供轻量重试机制即可，不强制额外按钮

## 7. 对话型大模型页面要求

- ChatGPT、Gemini、DeepSeek 等对话型页面中的不同对话页必须显示各自独立笔记
- 切换对话页后，笔记状态立即更新
- 不允许沿用上一对话页的笔记内容

## 8. 文案建议

推荐文案：

- 模块标题：`当前页笔记`
- 空状态：`暂无笔记，随手记下你的想法或待追问点`
- 保存中：`保存中...`
- 已保存：`已保存`
- 保存失败：`保存失败`
- 页面信息卡摘要：`笔记：暂无` / `笔记：已记录`

## 9. UI 约束

- 不引入复杂富文本工具栏
- 不在 V1 中加入过多辅助按钮
- 不让笔记区和高亮区永久等权并列占满页面
- 高亮列表和笔记输入区要有清楚的视觉分区

## 10. 对后续扩展的预留

当前 UI 方案要允许未来自然演进为：

- 笔记插入高亮引用
- 笔记导出
- 有笔记页面筛选
- 独立笔记视图
